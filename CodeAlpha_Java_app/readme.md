# Java Application with Gradle

## Project Overview

This project is developed as part of the CodeAlpha Internship and focuses on creating a Java application using Gradle as a build automation tool. The project demonstrates how Gradle can be used to automate the build process, manage dependencies, compile Java code, run tests, and generate application files.

The main objective of this project is to understand Java application development with Gradle and learn modern software build and automation practices.

## About Gradle

Gradle is an open-source build automation tool used for developing and managing software projects. It simplifies the development process by automating tasks such as compilation, testing, dependency management, and application packaging.

In this project, Gradle is used to organize the Java project structure and automate the complete build workflow.

## Working of the Application

The Java application is developed using Gradle project structure. The source code is placed inside the `src/main/java` directory, while test files are stored inside the test directory.

Gradle performs different tasks such as compiling Java code, managing required libraries, executing test cases, and generating build files. The application can be executed after successful completion of the Gradle build process.

## Gradle Commands Used

### Build the Project

```
gradle build
```

This command compiles the source code, runs tests, and creates build files.

### Run the Application

```
gradle run
```

This command executes the Java application.

### Run Tests

```
gradle test
```

This command executes the test cases and checks application functionality.

## How to Run the Project

1. Download or clone the repository.
2. Install Java JDK and Gradle.
3. Open the project folder.
4. Run the following command:

```
gradle build
```

5. Start the application:

```
gradle run
```

6. View the output in the terminal.
